    \item{\underline{\textbf{Theory:-}}} {\\
        \textbf{Difference Between XML and JSON}

        XML (eXtensible Markup Language) and JSON (JavaScript Object Notation) are two popular formats used for storing and exchanging data between systems. Both are platform-independent and human-readable, but they differ in structure, syntax, and usage.

        \textbf{XML (eXtensible Markup Language)}

        XML is a markup language designed to store and transport data. It uses custom tags to define data elements and supports attributes, namespaces, and document validation through DTD or XML Schema.

        Example:

        \begin{verbatim}
        <student>
            <name>Arijit</name>
            <age>21</age>
        </student>
        \end{verbatim}

        \textbf{JSON (JavaScript Object Notation)}

        JSON is a lightweight data-interchange format based on key-value pairs. It is easy for humans to read and write and simple for machines to parse and generate.
        \newpage
        Example:

        \begin{verbatim}
        {
            "name": "Arijit",
            "age": 21
        }
        \end{verbatim}

        \textbf{Comparison Between XML and JSON}

        \begin{table}[h]
        \centering
        \renewcommand{\arraystretch}{1.3}
        \begin{tabular}{|p{4cm}|p{5cm}|p{5cm}|}
        \hline
        \textbf{Feature} & \textbf{XML} & \textbf{JSON} \\
        \hline
        Full Form & eXtensible Markup Language & JavaScript Object Notation \\
        \hline
        Data Representation & Uses tags and attributes & Uses key-value pairs \\
        \hline
        Readability & More verbose and complex & Simpler and easier to read \\
        \hline
        File Size & Larger due to opening and closing tags & Smaller and compact \\
        \hline
        Parsing Speed & Slower & Faster \\
        \hline
        Data Types & Stores data as text; type conversion needed & Supports strings, numbers, arrays, booleans, objects, and null \\
        \hline
        Support for Comments & Supports comments & Does not officially support comments \\
        \hline
        Schema Validation & Supports DTD and XML Schema & Uses JSON Schema (optional) \\
        \hline
        Namespace Support & Supported & Not supported \\
        \hline
        Array Representation & More complex & Native array support \\
        \hline
        Human Readability & Less readable for large documents & More readable and concise \\
        \hline
        Web API Usage & Less common in modern APIs & Widely used in REST APIs \\
        \hline
        Processing Overhead & Higher & Lower \\
        \hline
        Popularity & Common in legacy systems & Preferred in modern web applications \\
        \hline
        \end{tabular}
        \caption{Comparison of XML and JSON}
        \label{tab:xmljson}
        \end{table}

        \textbf{Advantages of XML}

        \begin{itemize}
            \item Supports namespaces and document validation.
            \item Suitable for complex document structures.
            \item Platform and language independent.
            \item Extensible and self-descriptive.
        \end{itemize}

        \newpage
        \textbf{Advantages of JSON}

        \begin{itemize}
            \item Lightweight and compact.
            \item Faster parsing and processing.
            \item Easy to understand and write.
            \item Widely supported by modern programming languages and web APIs.
        \end{itemize}

        Both XML and JSON are widely used for data exchange. XML is suitable for complex documents requiring validation and namespaces, while JSON is preferred for modern web applications because of its simplicity, compact size, and faster processing. In contemporary software development, JSON has become the dominant format for web services and APIs.
