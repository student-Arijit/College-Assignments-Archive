\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{Loops in Java}
\end{center}
\vspace{0.5cm}

\noindent
Loops in Java are fundamental control structures that enable repeated execution of a block of code based on a specific condition. Rather than writing the same code multiple times, loops allow programmers to automate repetitive tasks efficiently and elegantly. Mastering loops is crucial for building logical constructs such as data processing, pattern generation, iterative calculations, and more.

Java provides three primary types of loops, each tailored to different use cases:

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Types of Loops in Java}]
\begin{itemize}[leftmargin=2em]
    \item \textbf{\texttt{for} loop} – Used when the number of iterations is predetermined. Ideal for counter-controlled loops.
    \item \textbf{\texttt{while} loop} – Condition is evaluated before each iteration. Suitable when the number of iterations is not known in advance.
    \item \textbf{\texttt{do-while} loop} – Executes the loop body at least once, since the condition is checked after the loop body.
\end{itemize}
\end{tcolorbox}

\vspace{0.5cm}

\subsection*{Example: Sum of First 5 Natural Numbers Using \texttt{for} Loop}
\begin{lstlisting}[language=Java,caption=Java Program to Find Sum of First 5 Natural Numbers,label=lst:sumForLoop]
public class LoopExample {
    public static void main(String[] args) {
        int sum = 0;
        for (int i = 1; i <= 5; i++) {
            sum += i;
        }
        System.out.println("Sum=" + sum);
    }
}
\end{lstlisting}

\noindent\textbf{Output:} \texttt{Sum = 15}

\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{Function Overloading in Java}
\end{center}
\vspace{0.5cm}

\noindent
Function Overloading in Java is a feature that allows a class to have more than one method with the same name, but with different parameter lists. It enhances readability and reusability of code and is a part of **compile-time (static) polymorphism**.

Overloaded methods must differ in the **number**, **type**, or **order** of parameters. Return type alone is not sufficient for method overloading.

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Rules of Function Overloading}]
\begin{itemize}[leftmargin=2em]
    \item Same method name but different parameter list.
    \item Method can differ by:
    \begin{itemize}
        \item Number of parameters
        \item Data type of parameters
        \item Order of parameters
    \end{itemize}
    \item Return type can be different but it’s not used to distinguish overloaded methods.
\end{itemize}
\end{tcolorbox}

\vspace{0.5cm}

\subsection*{Example: Function Overloading with Varying Parameters}
\begin{lstlisting}[language=Java,caption=Java Program Demonstrating Function Overloading,label=lst:functionOverloading]

 public int add(int a, int b) {
        return a + b;}public int add(int a, int b, int c) {return a + b +c;

%=======Array=======
\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{Arrays in Java}
\end{center}
\vspace{0.5cm}

\noindent
Arrays in Java are fundamental data structures that allow storage of multiple values of the same data type under a single variable name. They are useful for efficiently handling collections of data such as lists of numbers, names, or objects, and they simplify iteration and processing of related values. Each element in the array is identified by its index, starting from 0.

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Key Features of Arrays in Java}]
\begin{itemize}[leftmargin=2em]
    \item Arrays are indexed collections of fixed size.
    \item The index starts from 0 and goes up to (length - 1).
    \item Arrays can store primitive data types and objects.
    \item Declared with square brackets \texttt{[]}.
\end{itemize}
\end{tcolorbox}

\vspace{0.5cm}

\subsection*{Example: Storing and Printing Elements of an Integer Array}
\begin{lstlisting}[language=Java,caption=Java Program to Print Elements of an Array,label=lst:arrayIntro]
public class ArrayExample {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};
        System.out.println("Array elements are:");
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("Element atindex" + i + ": " + numbers[i]);
        }
    }
}
\end{lstlisting}

\noindent\textbf{Output:}
\begin{verbatim}
Array elements are:
Element at index 0: 10
Element at index 1: 20
Element at index 2: 30
Element at index 3: 40
Element at index 4: 50
\end{verbatim}

\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{OOP Concepts in Java}
\end{center}
\vspace{0.5cm}

\noindent
Object-Oriented Programming (OOP) is a paradigm that organizes software design around data, or objects, rather than functions and logic. Java is a pure object-oriented language (except for primitive types) and supports all major OOP principles. Mastering OOP helps in building scalable, reusable, and maintainable applications.

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Four Pillars of OOP in Java}]
\begin{itemize}[leftmargin=2em]
    \item \textbf{Encapsulation} – Binding data and code together within a class and restricting direct access to some components.
    \item \textbf{Inheritance} – Mechanism by which one class acquires the properties and behavior (methods) of another class.
    \item \textbf{Polymorphism} – Ability of a variable, function, or object to take multiple forms.
    \item \textbf{Abstraction} – Hiding complex implementation details and showing only the essential features.
\end{itemize}
\end{tcolorbox}

\vspace{0.5cm}

\subsection*{Example: Demonstrating Encapsulation in Java}
\begin{lstlisting}[language=Java,caption=Java Program to Demonstrate Encapsulation,label=lst:oopEncapsulation]
class Student {
    private String name;  // Private = Encapsulation

    // Getter method
    public String getName() {
        return name;  }
    // Setter method
    public void setName(String newName) {
        name = newName; }}
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();
        s1.setName("Alice");
        System.out.println("Student Name: " + s1.getName());}}
\end{lstlisting}

\noindent\textbf{Output:} \texttt{Student Name: Alice}

%=======Threading========
\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{Threading in Java}
\end{center}
\vspace{0.5cm}

\noindent
Threading in Java allows a program to perform multiple tasks concurrently, improving the efficiency and responsiveness of applications. Java provides built-in support for multithreading through the Thread class and the Runnable interface. These enable the creation and management of threads that can execute independently or cooperatively, depending on the design of the application.

A thread is a lightweight sub-process and the smallest unit of CPU execution. Unlike processes, threads share the same memory space, which allows for easy communication between them, but also introduces challenges such as synchronization and thread safety.

Multithreading is especially beneficial in situations such as:

Animations and GUI applications, where user interface responsiveness is critical.

File I/O operations, where reading and writing files can occur in the background without blocking the main thread.

Parallel computations, such as sorting large datasets or performing mathematical calculations concurrently.

Network operations, where waiting for data from remote sources can be handled asynchronously.

In Java, you can create a thread in two primary ways:

Extending the Thread class and overriding its run() method.

Implementing the Runnable interface, passing an instance of the class to a Thread object.

Additionally, Java provides high-level concurrency utilities in the javautil package, including ExecutorService, Callable, Future, and synchronization tools like locks, semaphores, and countdown latches.

Proper use of threading can greatly improve application performance, but it must be handled with care to avoid issues such as:

Race conditions

Deadlocks

Thread starvation

By understanding and applying multithreading principles effectively, developers can build robust, efficient, and responsive Java applications.

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Key Concepts in Java Threading}]
\begin{itemize}[leftmargin=2em]
    \item \textbf{Thread Class} – Inherit the \texttt{Thread} class and override the \texttt{run()} method.
    \item \textbf{Runnable Interface} – Implement \texttt{Runnable} and pass the object to a \texttt{Thread}.
    \item \textbf{start()} – Starts a new thread which calls the \texttt{run()} method internally.
    \item \textbf{run()} – Contains the code that should execute in a separate thread.
\end{itemize}
\end{tcolorbox}

\vspace{0.5cm}

%======JDBC========
\newpage
\begin{center}
    \Huge\bfseries \textcolor{black}{JDBC in Java}
\end{center}
\vspace{0.5cm}

\noindent
JDBC (Java Database Connectivity) is an API that allows Java programs to interact with relational databases. Using JDBC, Java applications can connect to a database, send SQL queries, and retrieve and manipulate data in a standardized way.

JDBC is part of the Java Standard Edition platform and works with all major databases like MySQL, Oracle, PostgreSQL, and SQLite.

\vspace{0.5cm}

\begin{tcolorbox}[colback=black!5!white,colframe=black,title=\textbf{Steps to Use JDBC in Java}]
\begin{enumerate}[leftmargin=2em]
    \item \textbf{Import JDBC packages:} \texttt{import java.sql.*;}
    \item \textbf{Load JDBC driver:} \texttt{Class.forName("com.mysql.cj.jdbc. Driver");}
    \item \textbf{Establish connection:} \texttt{Connection con = DriverManager.getConnection(...);}
    \item \textbf{Create statement:} \texttt{Statement stmt = con.createStatement();}
    \item \textbf{Execute query:} \texttt{ResultSet rs = stmt.executeQuery(...);}
    \item \textbf{Process results:} Use \texttt{while(rs.next())} to read data.
    \item \textbf{Close connection:} \texttt{con.close();}
\end{enumerate}
\end{tcolorbox}

\vspace{0.5cm}

\newpage
\subsection*{Example: Connecting to MySQL and Fetching Data}
\begin{lstlisting}[language=Java,caption=Java JDBC Program to Connect to MySQL,label=lst:jdbcExample]
import java.sql.*;

public class JDBCExample {
    public static void main(String[] args) {
        try {
            // Step 1: Load Driver
            Class.forName("com.mysql.cj.jdbc.Driver");

            // Step 2: Establish connection
            Connection con = DriverManager.getConnection(
                "jdbc:mysql://localhost:3306/mydatabase", "root", "password");

            // Step 3: Create statement
            Statement stmt = con.createStatement();

            // Step 4: Execute query
            ResultSet rs = stmt.executeQuery("SELECT * FROM students");

            // Step 5: Process results
            while (rs.next()) {
                System.out.println("ID: " + rs.getInt("id") +
                                   ", Name: " + rs.getString("name"));
            }

            // Step 6: Close connection
            con.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
\end{lstlisting}

\noindent\textbf{Note:}
\begin{itemize}[leftmargin=2em]
    \item Make sure to include the MySQL JDBC driver (e.g., \texttt{mysql-connector-
    java.jar}) in your project classpath.
    \item Replace database name, username, password, and table name with actual values.
\end{itemize}
